## Тема  
Серверная часть системы «Умная парковка»  

## План работ  
1. Оформление требований и архитектуры проекта  
2. Развёртывание и настройка серверной части (эмуляция Raspberry Pi)  
3. Разработка REST API для обмена с клиентом  
4. Интеграция клиента и сервера  
5. Тестирование и оптимизация системы  

## Требования  

### Сервер (Модель)  
1. Эмуляция данных от парковочных датчиков (например, занятость мест)  
2. Предоставление данных через REST API  
3. Приём запросов от клиента для получения статуса парковки  
4. Возможность изменения состояния парковочных мест (для тестов)  

### Клиент (Android приложение)  
1. Отображение доступных и занятых мест в реальном времени  
2. Получение данных с сервера  
3. Интерактивное обновление интерфейса при изменении состояния
